# User Stories for Buying a Car App - Consumer Version

## User Stories

1. **As a user, I want to be able to search for cars by make, model, year, and price range, so that I can find a car that fits my needs and budget.**

2. **As a user, I want to view detailed information about a car, including specifications, features, and images, so that I can make an informed decision.**

3. **As a user, I want to save my favorite cars to a wishlist, so that I can easily access them later.**

4. **As a user, I want to create an account and log in, so that I can manage my profile and saved cars.**

5. **As a user, I want to edit my profile information, including my contact details and preferences, so that my account is up to date.**

6. **As a user, I want to proceed to checkout and finalize my car purchase, so that I can buy the car I want.**

7. **As a user, I want to receive confirmation of my purchase via email, so that I have a record of the transaction.**

8. **As a user, I want to filter search results by various criteria such as mileage, fuel type, and transmission, so that I can narrow down my options.**

9. **As a user, I want to view reviews and ratings for cars, so that I can gauge the experiences of other buyers.**

10. **As a user, I want to be able to contact the seller directly through the app, so that I can ask questions about the car.**

## Additional Notes

- The application should be responsive and user-friendly, ensuring a smooth experience on both desktop and mobile devices.
- The design should incorporate Bootstrap for styling and layout, providing a modern and clean interface.
- User authentication should be secure, utilizing best practices for password storage and session management.
